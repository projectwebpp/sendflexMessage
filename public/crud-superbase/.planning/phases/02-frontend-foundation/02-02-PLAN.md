# Plan 02-02: Stores + React Router Layout

## Objective

Wire the React entry point (`main.jsx` with `BrowserRouter`), define the Routes tree (`App.jsx`), implement the two Zustand stores (`themeStore` persisting to localStorage, `uiStore` holding drawer state per D-03), and build the `Layout.jsx` Outlet wrapper with the mobile drawer overlay and backdrop-click-to-close behavior (D-01, D-02).

**Purpose:** Bridge the static scaffold (02-01) to the visual components (02-03). After this plan, the app routes between blank shells; after 02-03, those shells carry Sidebar + Topbar + page content.

**Output:** A routable React app where `/`, `/tasks`, `/analytics`, `/settings`, `/teams`, `/calendar` are wired through `<Layout />` `<Outlet />`, and the drawer state mechanism is in place (visual components consume it in 02-03).
